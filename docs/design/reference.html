What this is

The universal journal — consolidated line-item accounting across G/L, A/P, A/R, costing, and profitability.

ACDOCA replaced the ECC-era patchwork of totals tables with a single physical table holding every subledger, costing, and line-item posting. Every module posts here, once, with every dimension attached.

For analytics it is the single source of truth. For extraction it is the most unforgiving table in SAP — because the same structure that makes it powerful for reporting makes it lethal for naïve reads.

Extraction gotchas

!

Partitioning. Largest table in S/4HANA. Full-table reads time out. Always partition by RYEAR and POPER at minimum.

!

No SELECT *. Exhausts dialog work process memory in seconds. Use the released CDS view I_JournalEntryItem, or filter via SLT/ODP.

!

Licensing. SLT replication of ACDOCA requires Full Use License. ODP works under Runtime. Verify before committing to an architecture.
