SM59 (RFC Destination Configuration) is where SAP administrators configure RFC destinations: logical names for remote systems that want to connect via RFC. When you extract data using Python/pyrfc, ADF, or another tool, that external system must be registered in SM59 with a destination name, IP/hostname, port, client, and authentication method. Without SM59 configuration, RFC calls fail with "destination not found" errors.

In SM59, you define: **(1) destination name** (e.g., "EXTRACTION_PYTHON"), **(2) connection type** (TCP/IP for modern systems), **(3) RFC host/port** (SAP system IP and RFC port, typically 3200-3399), **(4) client/user credentials** (extract user must have appropriate authorizations), and **(5) options** (timeout, retry, SSL). Administrators configure SM59; data engineers invoke it by name.

For troubleshooting, if Python/pyrfc fails to connect, check SM59 first: does the destination exist? Is it active? Are credentials correct? SM59 also shows connection test results and activity logs. Most extraction tools abstract SM59 (you provide credentials directly in code), but understanding SM59 helps you troubleshoot network and authentication issues.
