Overview

SAP Datasphere Replication Flows is the cloud-native solution for SAP data integration. It automates data replication from on-premise SAP systems to cloud targets with automatic CDC via Resilient Data Buffer (RDB). Ideal for long-term cloud migration strategies.

Part 1: Setup & Enablement Check

Prerequisites

  • SAP Datasphere subscription (cloud)
  • SAP Cloud Connector (for on-premise source systems)
  • S/4HANA 2021 or later (on-premise source)
  • RFC resources configured: DHAMB_, DHAPE_, RFC_FUNCTION_SEARCH
  • Datasphere admin rights

Step 1: Install and Configure Cloud Connector

SAP Cloud Connector Setup:
  • Location: Download from SAP Launchpad
  • Install on: On-premise network DMZ/internal network
  • Configure to reach Datasphere region
  • Configure RFC protocol resources

Step 2: Create Connection in SAP Datasphere

UI Navigation: Connections → New Connection
Connection Type: SAP S/4HANA On-Premise
Configuration:
  • Name: e.g., "PROD_S4_Connection"
  • Cloud Connector: Select configured connector
  • System ID: Source SAP system ID
  • System Client: Client to replicate from
  • System Number: Source system number
  • Authentication: RFC user credentials
Validation: "Replication flows are enabled"

Step 3: Verify CDS View Configuration

In S/4HANA, create/verify CDS view: 
define view ZC_VBAK_EXTRACT
@Analytics.dataExtraction.enabled: true
@Analytics.dataExtraction.delta.changeDataCapture: true
@Analytics.dataExtraction.delta.changeDataCapture.automatic: true
as select from vbak {
  key vbak.vbeln,
  vbak.posnr,
  vbak.kunnr,
  vbak.erdat,
  vbak.netwr,
  ...
}
Verify all three annotations are present

Step 4: Enable Resilient Data Buffer (RDB)

In S/4HANA Source System:
  1. Transaction: /1DH/SETUP (Data Hub setup)
  2. Initialize Resilient Data Buffer framework
  3. Configure RFC user permissions for RDB
  4. Schedule observer job: /1DH/OBSERVE_LOGTAB
    • Frequency: Every 30 minutes
    • Purpose: Push changes to RDB buffers

Checking if Datasphere Replication is Enabled

Verification Step How to Check
Cloud Connector UI: https://localhost:8443 → Connected to Datasphere region
Datasphere Connection Connections → Test connection → "Connection successful"
CDS View Annotations SE11 → Verify all required annotations present
RDB Framework /1DH/SETUP → Verify active, check observer job scheduled

Part 2: Using Datasphere for VBAK Replication

Step 1: Create Replication Flow

In SAP Datasphere UI:
  1. Navigate: Data Builder → New Object → Replication Flow
  2. Flow Name: e.g., "VBAK_Replication_Flow"
  3. Description: Sales Document Header replication
  4. Source Connection: Select S/4HANA connection

Step 2: Select Source Object

Option 1: Direct CDS View (Recommended)
  • Container: CDS_EXTRACTION or SLT_EXTRACTION
  • Object Type: CDS View
  • Object Name: C_SALESDOCUMENTITEMDEX_1 (SAP standard)
    OR ZC_VBAK_EXTRACT (custom)
Option 2: SLT Table (If using SLT as source)
  • Container: SLT_EXTRACTION
  • Object Type: Table
  • Object Name: VBAK (if replicated via SLT)

Step 3: Configure Load Strategy

Initial Load Settings:
  • Load Type: "Initial and Delta"
  • Initial Load Mode: "Full Load"
  • Transfer Size: 100,000 records per batch
  • Parallel Processes: 4
  • Commit Strategy: Every 50,000 records
Delta Load Settings:
  • Enable: "Incremental Updates"
  • Delta Mode: "Automatic CDC" (Resilient Data Buffer)
  • Delta Frequency: Every 60 minutes (default, adjustable)
  • Handle Deletes: Yes

Step 4: Configure Target & Deploy

Target Selection:
  1. Target Container: e.g., "SALES_DATA" or "VBAK_TARGET"
  2. Create New Table: Yes
  3. Table Name: VBAK or C_VBAK_REPL
  4. Schema: Auto-mapped from source CDS view
Deployment:
  1. Click "Deploy" to validate flow
  2. Fix any validation errors
  3. Click "Run" to start replication

Datasphere Automatic Processes

Phase 1: Initial Load

  1. Datasphere calls Cloud Connector
  2. Cloud Connector executes RFC to S/4HANA
  3. S/4HANA reads VBAK via CDS view (4 parallel jobs)
  4. Batch size: 100,000 records per package
  5. Data flows: S/4HANA → CDS View → RFC → Cloud Connector → Datasphere
  6. Datasphere creates target table automatically
  7. Adds system columns: operation_flag, recordstamp, is_deleted

Phase 2: Delta Replication (Every 60 minutes by default)

  1. Database Triggers capture VBAK changes (I/U/D)
  2. Master Logging Table stores changes in S/4HANA
  3. Observer Job (/1DH/OBSERVE_LOGTAB) runs every 30 min
  4. Pushes changes to subscriber RDB buffers
  5. Replication Flow Delta Job calls RDB via RFC
  6. Retrieves changes from RDB buffer
  7. Datasphere applies changes:
    • INSERT: Add new row
    • UPDATE: Merge into existing row
    • DELETE: Mark with is_deleted=TRUE

Target Destinations

  • Datasphere Native: In-memory column-store tables
  • SAP Cloud: SAP HANA Cloud, SAP Analytics Cloud
  • Cloud Providers: Google BigQuery, Amazon S3/Redshift, Azure Data Lake/Synapse, Snowflake
  • Operational Systems: SAP S/4HANA, SAP C4C (via outbound flows)

Datasphere Target Table Columns

Column Type Details
Original VBAK Fields VBELN, POSNR, KUNNR, ERDAT, NETWR, WAERK, ...
operation_flag 'I' (Insert), 'U' (Update), 'D' (Delete)
recordstamp Timestamp of change - enables temporal queries
is_deleted TRUE = soft delete, FALSE = active record

Monitoring Replication

Monitor Location Purpose
Execution Status Datasphere: Replication Flow → Executions tab Last run, records processed, duration, status
Data Integration Datasphere: Data Integration Monitor All active flows, cumulative status, delta pipeline
RDB Status S/4HANA: /1DH/RSDBMON Buffer sizes, pending changes, observer job status
CDC Status S/4HANA: DHCDCMON Active streams, change rate, lag time

Quick Reference: Key Transactions

  • /1DH/SETUP - Resilient Data Buffer initialization
  • /1DH/OBSERVE_LOGTAB - Observer job for delta push
  • /1DH/RSDBMON - RDB buffer monitoring
  • DHCDCMON - Change Data Capture monitoring
