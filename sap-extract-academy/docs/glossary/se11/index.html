SE11: Data Dictionary / ABAP Dictionary in SAP

Definition

SE11 is the SAP transaction for the Data Dictionary. You use SE11 to view table definitions (fields, types, keys, append structures), understand field relationships (CUKY/CURR, MEINS/QUAN), and identify Z-fields before extraction.

SE11 (Data Dictionary / ABAP Dictionary) is the transaction where you explore table structures, field definitions, and metadata. When planning an extraction, your first step is SE11: open the table (ACDOCA, BKPF, VBAK), examine fields, identify primary keys, find CUKY/CURR and MEINS/QUAN pairings, and discover append structures (Z-fields). SE11 is your encyclopedia of table schemas.

In SE11, you see: field names, data types (CHAR, NUMC, CURR, DATS, QUAN), lengths, decimal places, primary key fields, foreign key relationships, and technical settings. For ACDOCA, you discover that RYEAR and POPER are primary key fields (partition strategy!), that AMOUNT_LOCAL pairs with LCURR (a CUKY field), and that multiple append structures may have been added over years. This information shapes your extraction architecture.

SE11 also shows table size (DBSTAT tab shows record count and physical size). For ACDOCA, SE11 might show 5 billion rows across all fiscal years—telling you this is an expert-level extraction requiring SLT and parallelism. SE11 is where you become table-literate before writing any extraction code.
