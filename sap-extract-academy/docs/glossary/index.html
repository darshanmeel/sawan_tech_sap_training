ABAP

Advanced Business Application Programming

ABAP is SAP's proprietary programming language used to develop custom applications, reports, interfaces, and enhancements within the SAP ecosystem. It runs on the SAP application server and accesses SAP databases directly.

View

ADT

ABAP Development Tools

ADT is the modern IDE for ABAP development, built on Eclipse. It replaces the classic SAP GUI-based development tools (SE38, SE11) with a native IDE offering syntax highlighting, refactoring, debugging, and Git integration.

View

Append Structure

Append Structure

An append structure is a SAP table extension mechanism that allows customers to add custom fields (Z-fields) to a standard SAP table without modifying the table directly. Multiple append structures can be stacked on one table.

View

BAdI

Business Add-In

A BAdI is SAP's plug-in architecture that allows custom code to hook into standard business logic without modifying source code. Used for post-processing, filtering, or extending transactions and batch jobs.

View

BW

SAP Business Warehouse

SAP BW is SAP's legacy business intelligence and data warehousing solution. Now superseded by S/4HANA Embedded Analytics and cloud solutions (SAP Analytics Cloud), but still used for enterprise reporting in many organizations.

View

CDS

Core Data Services

CDS is SAP's declarative language for defining data models and views. CDS views are the modern, supported way to expose SAP data; they enforce authorization, apply currency conversion, and avoid raw table queries.

View

CUKY

Currency Key

CUKY is a SAP data type field that stores a currency code (e.g., USD, EUR, JPY). Often paired with amount fields; you must extract both CUKY and its corresponding amount to properly interpret the value.

View

CURR

Currency Amount

CURR is a SAP data type for numeric fields representing monetary amounts. A CURR field must be paired with a CUKY (currency key) field to be interpretable; extracting one without the other creates orphaned data.

View

DATS

Date (8-digit format)

DATS is SAP's date data type stored as YYYYMMDD (8 digits). Example: 20260422 for April 22, 2026. SAP handles DATS-to-local-date conversion automatically, but extractors must preserve the format.

View

Delta

Delta (Change Data Capture)

Delta represents only the rows that have changed since the last extraction. Real-time delta captures inserts/updates/deletes immediately; queue-based delta periodically batches changes. Enables streaming architectures without full table replication.

View

Dialog Work Process

Dialog Work Process

A dialog work process is an SAP application server process that handles interactive user sessions (SE16N, MM01, FBL3N). When you slow down extraction, you risk blocking dialog users; use non-dialog processes (batch) for heavy extraction work.

View

Extractor

Data Extractor

An extractor is a reusable SAP component that exposes table data for extraction via ODP or custom querying. Generic extractors exist for all tables; they support filtering, field selection, and delta tracking.

View

LTRC

SLT Landscape Transformation Replication Client

LTRC is the SAP GUI transaction for SLT configuration on the source SAP system. You define replication objects, partitioning, filters, and field selections in LTRC; it's the control interface for SLT replication.

View

LTRS

SLT Landscape Transformation Replication Server

LTRS is the SAP component that runs parallel reader processes for SLT replication. You configure LTRS to run 4-8 parallel readers, each handling a partition (e.g., one company code + fiscal year), dramatically accelerating full-load times.

View

MANDT

Client / Mandant

MANDT is a 3-digit field present in every SAP table that identifies which client (tenant) the row belongs to. Multi-client systems use MANDT to isolate data. Always filter MANDT in extraction to ensure correct client data.

View

MEINS

Material Unit of Measure

MEINS is a field in MARA (material master) that specifies the base unit of measure for a material: EA (each), KG (kilogram), LT (liter), CM (centimeter), etc. Must be paired with quantity fields when extracting.

View

NUMC

Numeric Character String

NUMC is a SAP data type for numeric strings with leading zeros preserved (e.g., '00001234'). Common for codes and IDs that must display leading zeros but are treated as numbers in SAP.

View

ODP

Operational Data Provisioning

ODP is SAP's extraction framework for exposing table and view data for real-time extraction. ODP handles authorization, supports full-load and delta, and integrates with CDS views. The modern alternative to legacy BW extractors.

View

ODQMON

ODP Queue Monitor

ODQMON is the SAP transaction for monitoring ODP delta queues. Shows extraction status, delta lag, queue depth, and any errors. Essential for troubleshooting slow delta extractions or queue buildup.

View

OpenHub

SAP BW OpenHub

OpenHub is SAP's legacy data distribution mechanism for exporting BW cube data to external systems. Now largely replaced by ODP and modern cloud APIs, but still used in older BW implementations.

View

QUAN

Quantity

QUAN is a SAP data type for numeric quantity fields (order quantities, inventory balances, weights). Like CURR, QUAN fields must be paired with unit fields (MEINS or UOOM) to be interpretable.

View

RFC

Remote Function Call

RFC is SAP's protocol for remote procedure calls between systems. Python/pyrfc libraries use RFC to call extraction functions (RFC_ODP_READ, RFC_READ_TABLE) without logging into SAP GUI directly.

View

S/4HANA

SAP S/4HANA (SAP Suite 4 High-Performance Analytics)

S/4HANA is SAP's latest ERP platform, released in 2015. It replaces older ERP versions (R/3, ECC) with modern architecture: in-memory database (HANA), cloud-first design, simplified data models, and embedded analytics.

View

SAP Help Portal

SAP Help Portal

The SAP Help Portal (help.sap.com) is the official SAP documentation site. Contains technical guides, API references, transaction documentation, and troubleshooting for all SAP products. Essential resource for extraction work.

View

SE11

Data Dictionary / ABAP Dictionary

SE11 is the SAP transaction for the Data Dictionary. You use SE11 to view table definitions (fields, types, keys, append structures), understand field relationships (CUKY/CURR, MEINS/QUAN), and identify Z-fields before extraction.

View

SE16N

Table Data Browser (Generic)

SE16N is the SAP transaction for browsing table data with filtering and sorting, replacing legacy SE16. You use SE16N to count rows, verify data quality, and manually test filter conditions before automating extraction.

View

SICF

HTTP/HTTPS Configuration

SICF is the SAP transaction for configuring HTTP handlers, services, and web communication. Used to enable REST APIs, iGate services, or custom web handlers that extraction tools might call.

View

SLT

SAP Landscape Transformation

SLT is SAP's licensed tool for high-performance data replication from SAP to external systems (Kafka, databases, data lakes). Enables billions-of-rows extractions with real-time delta, partitioning, and parallel reads.

View

SM59

RFC Destination Configuration

SM59 is the SAP transaction for configuring RFC destinations (connections to external systems or remote SAP systems). You configure SM59 to enable RFC calls from Python, ADF, or other tools to extract data.

View

Transport Request

Transport Request / Change Request

A transport request is SAP's mechanism for moving code, configuration, and customizations across environments (dev → test → prod). Every custom ABAP object or CDS view exists in a transport request to track changes.

View

TSV_TNEW_PAGE_ALLOC_FAILED

SAP Memory Allocation Error

TSV_TNEW_PAGE_ALLOC_FAILED is an SAP error indicating memory exhaustion during extraction. Common when SELECT *-ing large tables (ACDOCA, BKPF) without filters. Solution: use CDS views, apply filters, or use SLT for parallelism.

View

VBELN

Sales and Distribution Document Number

VBELN is the primary key field for sales documents (VBAK table). It's a 10-digit document number that uniquely identifies a sales order, delivery, invoice, or return. Common extraction partition key.

View

Z-field

Custom Field (Z-prefix)

Z-fields are custom fields added by customers to SAP tables via append structures. Named Z* or Y* by convention, they represent enterprise-specific logic not provided by standard SAP. Must be explicitly extracted or data warehouse is incomplete.
