VBAK

Sales Document Header

Header record for sales documents (orders, quotations, contracts). One row per sales document.

What this table is

VBAK is the sales document header table in SAP S/4HANA, storing the master data for all sales documents: purchase orders, quotations, contracts, and returns. Each row represents one sales document with 100+ fields covering dates, organizational context, document type, and status.

The table is fundamental to order-to-cash (O2C) analytics. It links to VBAP (line items), VBKD (sales document schedule lines), and other SD tables. Most extraction use cases start with VBAK because it's the entry point for understanding sales volume, velocity, and customer patterns.

SAP S/4HANA Sales documents overview provides the complete technical reference. For cloud customers, SAP strongly recommends the released CDS view I_SalesDocument, which enforces authorization and applies post-processing logic automatically. Raw VBAK extraction bypasses these safeguards and should only be used when business rules don't apply.

Extraction gotchas

  • Sales document type (AUART) drives business logic. Filtering by type during extraction improves performance and ensures correct subset. SAP Note (opens in new tab)
  • Large enterprises partition VBAK by sales organization (VKORG) and document type to manage extraction windows. Consider this in your delta strategy. SAP Note (opens in new tab)
