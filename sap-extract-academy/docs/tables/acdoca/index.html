ACDOCA

Universal Journal

Universal Journal (subledger, costing, and line-item accounting). Billions of rows; consolidated view of all postings across all document types.

What this table is

ACDOCA is the Universal Journal table in SAP S/4HANA, storing consolidated line-item accounting postings across all modules: G/L, A/P, A/R, costing, and profitability. The table is often the largest in S/4HANA installations, with billions of rows accumulated over years. It's the source of truth for financial analytics, audit trails, and balance sheet reconciliation.

ACDOCA extraction is the centerpiece of data warehouse projects but requires disciplined engineering because naive "SELECT *" queries crash SAP systems. Successful extraction depends on partitioning strategy (by fiscal year and period), use of released CDS views, and careful load balancing across parallel jobs.

SAP S/4HANA Universal Journal overview documents the complete design. The ACDOCA architecture guide is mandatory reading if you're extracting more than a few years of data. Always use I_JournalEntryItem (the released CDS view) rather than raw ACDOCA — it enforces authorization, applies currency conversion, and protects against accidental full-table scans.

Extraction gotchas

  • ACDOCA is the largest table in S/4HANA. Extracting the entire table will timeout. Always partition by RYEAR (reporting year) and POPER (period) at minimum. SAP Note (opens in new tab)
  • Raw SELECT * on ACDOCA crashes SAP dialog work processes due to memory exhaustion. Use the released CDS view I_JournalEntryItem instead, or apply row-level filtering with SLT/ODP. SAP Note (opens in new tab)
  • ACDOCA consolidates all accounting: G/L, A/P, A/R, costing, profitability. Each posting appears once with multiple dimensions. Aggregation logic is business-rule dependent. SAP Note (opens in new tab)
  • Full Use License required for SLT replication of ACDOCA. ODP extraction (recommended) works under Runtime License. Verify your licensing with SAP before committing to SLT. SAP Note (opens in new tab)
