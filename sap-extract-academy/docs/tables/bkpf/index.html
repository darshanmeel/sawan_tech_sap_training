What this is

Accounting document header (all G/L, A/P, A/R documents). One row per accounting document, with posting date, document type, company code, fiscal year.

BKPF is the accounting document header table in the Financial Accounting (FI) module, storing metadata for all posted and parked accounting entries: posting dates, document types, company codes, fiscal years, amounts, and approval status. Each accounting document has one BKPF record (identified by BUKRS, GJAHR, BELNR — company code, fiscal year, document number).

Extracting BKPF is central to financial analytics and audit trails. The table is large (tens to hundreds of millions of documents annually in large enterprises) and requires careful partitioning by fiscal year and company code to avoid extraction timeouts.

SAP S/4HANA Accounting Document overview documents the structure. Note that BKPF is header-only (no monetary amounts broken down by account). For account-level details, join to BSEG (line items). Most data warehouse fact tables combine BKPF header context with BSEG line detail.

Key fields

Field Role Description
MANDT Key Client (always filter to current client, never extract all)
BUKRS Key Company Code (4-char, primary partition key)
GJAHR Key Fiscal Year (partition key; most extractions slice by year)
BELNR Key Accounting Document Number (10-char, unique within BUKRS+GJAHR)

Extraction gotchas

!

BKPF includes posted and parked documents. Filter BUKRS (company code) and GJAHR (fiscal year) early to reduce volume. Most extractions partition by fiscal year. SAP Note ↗(opens in new tab)

!

Line items live in BSEG, not BKPF. BKPF is header only (amounts, posting date, approver). Join to BSEG for account-level details. SAP Note ↗(opens in new tab)

!

Accounting documents can be reversed. Check AUGBL (reversal reference) and STORNOMON (reversal flag) to understand reversals vs. new postings. SAP Note ↗(opens in new tab)
