📞 RFC - Remote Function Call

Traditional SAP Protocol for Internal System-to-System Communication

← Back to Main
← Back to All Methods Legacy Support Internal SAP 1-5 min Latency
⚠️ Important Policy Update (SAP Note 3255746):
Native ODP RFC modules are now "UNPERMITTED" for third-party applications. SAP recommends using ODP Framework or OData Services for external integrations. RFC remains essential for internal SAP-to-SAP connections (like SLT).

Overview

RFC is the traditional SAP protocol for synchronous function calls between systems. While still used for internal SLT-based replication, SAP has restricted its use for third-party applications. Best for legacy systems and internal SAP-to-SAP integration.

Part 1: Setup & Enablement Check

Prerequisites

  • SAP source system with RFC modules active
  • SAP target system or SLT Replication Server
  • Network connectivity and firewall rules configured
  • RFC users with appropriate authorizations

Step 1: Create RFC Connection

Transaction: SM59
Connection Type: 3 (ABAP to ABAP)
Details:
  • Connection Name: e.g., "PROD_VBAK_SLT"
  • Target Host: Source SAP hostname
  • System Number: Source system number
  • Client: Source client (e.g., 100)
  • Logon User: RFC communication user
  • Query Timeout: 600 seconds

Step 2: Test RFC Connection

Transaction: SM59
  1. Select your RFC connection
  2. Click "Connection Test"
  3. Click "Remote Logon Test"
  4. Verify: "Connection OK" message
  5. Check user permissions sufficient

Step 3: Authorize RFC User

Transaction: SU01
Authorization Objects Required:
  • S_DEVELOP - Table access for extraction
  • S_TABU_DIS - VBAK table access (full access)
  • RFC1 - RFC call permission
  • S_RFC - RFC function module execution
Steps:
  1. User: Select RFC user account
  2. Assign roles: SAP_USER (basic) + Z_RFC_VBAK_ACCESS (custom)
  3. Save and confirm

Checking if RFC is Enabled

Verification Step How to Check
RFC Connection Status SM59 → Select connection → Connection Test
RFC User Permissions SU01 → User → Check S_TABU_DIS for VBAK
Network Connectivity SM59 → Connection test should pass
SLT RFC Config LTRC → Configuration → Verify RFC selected

Part 2: Using RFC for VBAK Replication (via SLT)

Step 1: Configure Full Load Transfer

Transaction: LTRC
  1. Select VBAK table in Data Provisioning
  2. Set "Initial Load Mode": Performance Optimized
  3. RFC Transfer Parameters:
    • Data Transfer Jobs: 4
    • Package Size: 50,000 records
    • RFC Timeout: 600 seconds
    • Retry Count: 3
  4. Click "Start Replication (Including Initial Load)"

Step 2: Configure Delta Load Transfer

  1. After initial load completes
  2. SLT monitors logging table via RFC calls
  3. Retrieves delta batches periodically
  4. Sends updates through RFC interface
  5. Frequency: Default every 5,000 records (1-5 minutes)

RFC Automatic Processes

Phase 1: Initial Load

  1. SLT constructs RFC function calls
  2. Calls SM59 connection to source system
  3. Source system reads VBAK in parallel (4 jobs)
  4. Data packaged in 50,000-record batches
  5. RFC transfer sends each batch to target
  6. Target writes received data
  7. Acknowledgment sent back via RFC
Performance: Typically 100K-500K records/minute

Phase 2: Delta Load

  1. Source system maintains logging table (VBAK#0)
  2. SLT queries logging table via RFC
  3. Retrieves batch of deltas (5,000 records)
  4. Identifies change type (I/U/D)
  5. Sends delta batch via RFC to target
  6. Target applies changes
  7. Process marks records as processed

RFC Target Destinations

  • SAP Systems: SAP HANA, SAP BW, SAP S/4HANA, Legacy SAP R/3
  • Custom ABAP Systems: Any system with ABAP stack and RFC support
  • Via SLT to Cloud: SAP Data Intelligence, Cloud databases, AWS/Azure/GCP

RFC Logging Table (VBAK#0)

Field Description
VBELN Sales document number (key)
POSNR Item number (key)
OPERATION I (Insert), U (Update), D (Delete)
TMSTMP Timestamp of change

Monitoring RFC Replication

Monitor Transaction What to Check
Data Transfer Monitor LTRC RFC call status, records transferred, timeouts
RFC Connection SM59 Connection state, last successful call
Logging Table SE12 VBAK#0 row count, unprocessed changes

Quick Reference: Key Transactions

  • SM59 - Create/test RFC connections
  • SU01 - User authorization configuration
  • LTRC - SLT Cockpit (for RFC-based replication)
