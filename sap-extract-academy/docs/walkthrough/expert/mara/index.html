Getting started

Scenario

Your company's MARA is heavily extended with custom fields. Manually managing 20+ Z-field extractions is error-prone. Fivetran's Application Layer connector discovers and extracts everything automatically.

What you've built

You have a complete material dimension in Snowflake, including all Z-fields and related plant/sales org data.

0 of 5 steps complete
  1. Step step-01

    Profile the extended MARA structure

    Your MARA has 20+ Z-fields across multiple append structures. Fivetran's Application Layer can extract all at once.

    Verify

    Count Z-fields in SE11 MARA → Append Structures.

  2. Step step-02

    Set up Fivetran Connector for SAP

    In Fivetran, create connector for SAP Application Layer. Authenticate with EXTRACT user.

    Verify

    Fivetran dashboard shows SAP connection status: Connected.

  3. Step step-03

    Enable MARA, MARC, MVKE tables in Fivetran

    Select all three tables. Fivetran auto-discovers joins and Z-fields.

    Verify

    Fivetran shows 3 tables with all Z-fields listed.

  4. Step step-04

    Configure Snowflake destination

    Map to Snowflake schemas: raw.MARA, raw.MARC, raw.MVKE.

    Verify

    Initial load completes.

  5. Step step-05

    Create Snowflake views with joins

    Create DIM_MATERIAL_COMPLETE joining MARA + MARC + MVKE + all Z-fields.

    Verify

    View query works and returns denormalized material records.

Troubleshooting

Fivetran transformation fails due to Z-field type mismatch

In Fivetran, adjust column mapping to handle string/numeric conversions.
