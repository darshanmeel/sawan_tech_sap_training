Getting started

Scenario

Your finance team needs accounting documents in the data lake for reconciliation and analytics. BKPF is large, so you'll start small: one company code, one fiscal year. This teaches you the partitioning pattern you'll use for multi-year, multi-company extractions later.

Estimated time: 50 minutes.

What you've built

You've extracted BKPF using a partitioned approach. You now understand that financial tables require careful partitioning by fiscal year and company code. The folder structure (finance/accounting/full-load/fy[YEAR]/bukrs[CODE]/) is repeatable — when 2025 arrives, you'll clone this pipeline, adjust filters, and extract new data to the same structure.

0 of 7 steps complete
  1. Step step-01

    Identify the I_AccountingDocument CDS view

    I_AccountingDocument is the released CDS view for BKPF.

    Do this in SAP

    Open transaction SE80 (opens in new tab) and navigate to Repository Browser → I_AccountingDocument.

    Verify

    View exists with @Analytics.dataExtract: true.

  2. Step step-02

    Understand BKPF partitioning (BUKRS, GJAHR)

    BKPF is large. The primary key is MANDT, BUKRS (company code), GJAHR (fiscal year), BELNR (document number). Always partition extractions by BUKRS and GJAHR to avoid memory exhaustion.

    Do this in SAP

    Open transaction SE16N (opens in new tab) and navigate to Execute → BKPF.

    Verify

    You see rows with BUKRS='1000' and GJAHR='2024' (or latest year). Count them — this is your expected row count for one partition.

  3. Step step-03

    Create ADF source dataset with BUKRS and GJAHR filters

    Create the I_AccountingDocument source dataset. Add parameters for BUKRS and GJAHR so you can partition the extraction. For this beginner walkthrough, fix them to one company code and one fiscal year.

    Verify

    Preview shows documents from company code 1000, fiscal year 2024 only.

  4. Step step-04

    Create ADLS folder structure

    In sap-raw, create finance/accounting/full-load/fy2024/bukrs1000/.

    Verify

    Folder structure exists.

  5. Step step-05

    Create sink dataset and pipeline

    Sink to ADLS Parquet under finance/accounting/full-load/fy2024/bukrs1000/. Build pipeline with Copy activity.

    Verify

    Pipeline succeeds.

  6. Step step-06

    Verify document count matches SAP

    Count rows in the Parquet file. Cross-check with SE16N (BUKRS=1000, GJAHR=2024) row count.

    Verify

    Row count matches. Sample documents show posting dates in 2024, company code 1000.

  7. Step step-07

    Document the partitioning strategy

    For future fiscal years or additional company codes, clone this ADF pipeline and adjust the BUKRS and GJAHR filters. Document the folder structure so downstream users understand the partition scheme.

    Verify

    Runbook or README in ADLS documents: 'finance/accounting/full-load/fy[YEAR]/bukrs[CODE]/'.

Troubleshooting

Filter condition doesn't work in ADF source dataset

In ADF, the ODP filter syntax is case-sensitive and space-sensitive. Use BUKRS='1000' and GJAHR=2024 (no quotes on numeric GJAHR).

Extraction times out despite filter

Even one company code + one fiscal year can be large (100M+ rows). If timeout persists, partition further by BLART (document type) or BUDAT (posting date).
