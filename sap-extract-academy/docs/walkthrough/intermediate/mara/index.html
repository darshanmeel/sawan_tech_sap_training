Getting started

Scenario

Your company extended MARA with Z-fields for brand and sustainability tracking. You need these in the data warehouse without modifying the released CDS view.

What you've built

You've extracted MARA with Z-fields via a safe, maintainable CDS extension view. The data warehouse now includes brand and sustainability dimensions.

0 of 5 steps complete
  1. Step step-01

    Identify Z-fields in MARA

    Your MARA has Z-fields: ZZ_BRAND (20 char), ZZ_SUSTAINABILITY (1 char). Confirm in SE11.

    Do this in SAP

    Open transaction SE11 (opens in new tab) and navigate to Data Dictionary → Tables → MARA → Fields → Find Z*.

    Verify

    ZZ_BRAND and ZZ_SUSTAINABILITY appear in MARA.

  2. Step step-02

    Create CDS extension view for Z-fields

    In SE80, create a CDS extension view ZPROD_ZFIELDS that adds Z-fields to I_Product without modifying the released view.

    Verify

    View ZPROD_ZFIELDS appears in SE80 with @Analytics.dataExtract: true.

  3. Step step-03

    Extract via the extension view

    Use pyrfc to extract from ZPROD_ZFIELDS instead of I_Product. This includes Z-fields safely.

    Verify

    Python script successfully reads ZZ_BRAND and ZZ_SUSTAINABILITY values.

  4. Step step-04

    Land in Snowflake with Z-field columns

    Parquet schema now includes ZZ_BRAND and ZZ_SUSTAINABILITY. Store in Snowflake MATERIAL table.

    Verify

    SELECT ZZ_BRAND, ZZ_SUSTAINABILITY FROM MATERIAL returns expected values.

  5. Step step-05

    Document Z-field lineage

    In your data dictionary, document: MARA.ZZ_BRAND → MATERIAL.ZZ_BRAND, MARA.ZZ_SUSTAINABILITY → MATERIAL.ZZ_SUSTAINABILITY. Note that these are append-structure fields (custom, not part of standard table).

    Verify

    Data dictionary includes Z-field lineage.

Troubleshooting

CDS extension view won't save (syntax error)

Ensure @AccessControl.authorizationCheck is present. Test in SE80 with Ctrl+F12.
