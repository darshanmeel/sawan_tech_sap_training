intermediate ACDOCA

Extract ACDOCA One Company/Year via ODP (Intermediate)

Partial ACDOCA extraction (one company code, current fiscal year) via ODP and the released CDS view I_JournalEntryItem. Avoids raw table pitfalls. Python/Snowflake.

Estimated time
90 minutes
Destinations
Snowflake
Extractors
Python

License check

All licenses. ODP is application-layer, works under Runtime License.

Overview

Scenario

Your finance team needs ACDOCA data for year-end close analytics. You'll extract one company, one fiscal year — enough for meaningful analysis without hitting system limits. This walkthrough teaches the safe approach; the Expert walkthrough scales to multi-year, multi-company via SLT.

Estimated time: 90 minutes (depends on partition size).

What you've built

You've successfully extracted ACDOCA without crashing SAP. You used the released CDS view (which every SAP professional should), streamed data to avoid memory exhaustion, and reconciled totals. This is the correct, professional approach to ACDOCA extraction.

When you need more years or companies, the Expert walkthrough scales these patterns via SLT and Kafka.

Before you start

  • Completed any Intermediate walkthrough
  • Understanding of ACDOCA size/complexity from table overview
0 of 5 steps complete
  1. Step 1

    Confirm ACDOCA is massive

    Use ACDOCA SE16N to count rows in one company code for one fiscal year. Budget 2-4 hours for extraction if rows exceed 100M.

    Do this in SAP
    Open transaction SE16N (opens in new tab) and navigate to Execute → ACDOCA, filter RBUKRS='1000' AND RYEAR=2024.
    Verify
    You know the row count for your partition. If > 500M, move to Expert walkthrough.
  2. Step 2

    Use I_JournalEntryItem CDS view (NOT raw ACDOCA)

    Raw SELECT * on ACDOCA crashes SAP. Always use I_JournalEntryItem, which enforces authorization and applies currency conversion.

    Do this in SAP
    Open transaction SE80 (opens in new tab) and navigate to Search → I_JournalEntryItem.
    Verify
    CDS view exists with @Analytics.dataExtract: true.
  3. Step 3

    Extract with fiscal year + company code filters

    Python/pyrfc with ODP. Filter: RBUKRS='1000' AND RYEAR=2024. Set timeouts high (60 min) because ACDOCA is slow even with filters.

    Verify
    Extraction completes. Logs show 'Fetched X million rows'.
  4. Step 4

    Stream to Snowflake to avoid memory issues

    Don't load all rows into Python memory. Stream in batches of 100K rows, upsert to Snowflake in micro-batches.

    Verify
    Snowflake table fills incrementally. Monitor via Snowflake query history.
  5. Step 5

    Reconcile with BKPF totals

    Sum amounts in ACDOCA (by company, year) and compare to BKPF aggregates. They should match (within rounding).

    Verify
    Totals match within 0.01%.

Troubleshooting

ODP extraction times out after 2 hours

ACDOCA partition is > 500M rows. Use the Expert walkthrough with SLT instead.

I_JournalEntryItem CDS view is slow (> 200M rows)

Add posting period filter: AND POPER >= '001' AND POPER <= '012' to further narrow range.
