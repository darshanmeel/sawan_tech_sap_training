Getting started

Scenario

Your procurement dashboard needs hourly vendor updates. Weekly full load with daily/hourly deltas is the pattern.

What you've built

You have a tiered extraction: weekly full reconciliation + hourly incremental updates. Vendor master is now near-real-time.

0 of 5 steps complete
  1. Step step-01

    Plan full + delta strategy

    Weekly full load (Sunday) + hourly delta (Mon-Sat). This balances freshness with extraction overhead.

    Document: Full load every Sunday 22:00 UTC, delta every hour 00-23:00 UTC.

  2. Step step-02

    Create ODP subscription for I_Supplier

    Use pyrfc to create a persistent ODP subscription for delta tracking.

    Subscription appears in ODQMON.

  3. Step step-03

    Implement full load (nightly)

    Python script with pyrfc for full load. Store to Snowflake STAGING_LFA1.

    Full load completes and STAGING_LFA1 is populated.

  4. Step step-04

    Implement delta extraction (hourly)

    Separate script that calls ODP delta API, stores sequence number in metadata table, applies changes to production LFA1 table via MERGE.

    Hourly delta runs and updates production table.

  5. Step step-05

    Monitor delta reconciliation

    Weekly, compare row count: full load count should match production count. Investigate deltas if they diverge.

    Count(STAGING) ≈ Count(LFA1_PROD)

Delta misses some updates

ODP queue may have dropped entries if extraction was delayed >2 days. Re-run full load to catch missing data.
