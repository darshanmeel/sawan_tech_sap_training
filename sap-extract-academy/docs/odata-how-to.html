Overview

OData Services expose data through REST/HTTP APIs, making them ideal for cloud platforms and external systems. While slower than native protocols, OData offers firewall-friendly connectivity and broad platform support.

Part 1: Setup & Enablement Check

Prerequisites

  • SAP Gateway system (NetWeaver or SAP Fiori)
  • OData provider services exposed
  • Client applications with HTTP/HTTPS support
  • Web service user credentials

Step 1: Expose ODP Provider as OData Service

Transaction: /IWFND/MAINT_SERVICE
  1. Access SAP Gateway
  2. Create/register service for your provider
  3. Service Type: OData (V2 or V4)
  4. Select Provider:
    • CDS View (e.g., Z_VBAK_EXTRACT)
    • Standard DataSource (2LIS_11_VAHDR)
    • BW query
  5. Activate Service
  6. Set Access Level: Public (if external access needed)

Step 2: Register OData Service Metadata

  1. Service alias: /sap/opu/odata/sap/2LIS_11_VAHDR_CDS
  2. Document service endpoint
  3. Generate technical documentation
  4. Create user documentation

Step 3: Create Gateway User

Transaction: SU01
  • User type: System/Communication
  • Role: SAP_GATEWAY_USER
  • Assign: VBAK table access
  • Set password: Secure credentials

Checking if OData Service is Enabled

Verification Step How to Check
Service Registration /IWFND/MAINT_SERVICE → Search service → Status = "Active"
Service Metadata URL: Service + /$metadata → Returns valid XML schema
Test Access Postman/curl → GET Service/Entity → HTTP 200
Gateway User SU01 → User → Verify active with roles assigned

Part 2: Using OData for VBAK Replication

Step 1: Configure OData Consumer

In External System (e.g., Amazon AppFlow):
  1. Create new data source
  2. Type: SAP OData Service
  3. URL: https://[SAP_HOST]:port/sap/opu/odata/sap/2LIS_11_VAHDR_CDS
  4. Authentication:
    • Type: Basic Auth
    • Username: Gateway user
    • Password: User password
  5. Test connection

Step 2: Configure Full Load Extraction

  1. Entity: ZMY_VBAK_SET (VBAK data)
  2. Load Mode: Full Extract
  3. Fields to Extract:
    • VBELN (Sales Document Number)
    • POSNR (Item Number)
    • KUNNR (Customer Number)
    • ERDAT (Creation Date)
    • NETWR (Net Value)
    • WAERK (Currency)
  4. Filter (optional): $filter=ERDAT ge '20260101'
  5. Pagination: $top=100000, $skip=0 (repeat for all records)

Step 3: Configure Delta Load Extraction

  1. After full load completes
  2. Delta Detection: ODP change tokens
  3. Schedule: Every 15 minutes (minimum)
  4. Delta Query:
    • Include timestamp filter
    • Request only modified records
    • Use change data capture fields
  5. Target Merge: Upsert (update/insert)

OData Automatic Processes

Phase 1: OData Full Load

HTTP Request Pattern: 
GET /sap/opu/odata/sap/2LIS_11_VAHDR_CDS/ZMY_VBAK_SET
Headers: Authorization, Accept: application/json
Process:
  • Returns first 10,000 records (default page size)
  • Includes pagination token
  • Consumer repeats request with token until completion

Phase 2: OData Delta Load (Every 15 minutes)

GET /sap/.../ZMY_VBAK_SET?$filter=LastModified gt 'timestamp'
Response includes:
  • Records modified since timestamp
  • ODQ_CHANGEMODE: 'C' (change) or 'D' (delete)
  • Last modified timestamp
  • Record version ID

OData Target Destinations

  • Analytics: Amazon AppFlow, Azure Data Factory, Google Cloud Data Integration
  • Platforms: Qlik, Tableau, Snowflake, Apache Spark
  • Data Lakes: S3, Azure Blob, GCS
  • Integration: Fivetran, Talend, Informatica, Stitch

OData Delta Load Fields

Field Description
ODQ_CHANGEMODE "C" = Change/Insert/Update, "D" = Delete marker
ODQ_ENTITYCNTR Unique counter per change, prevents duplicates
Original VBAK Fields All extracted fields (for deletes: key fields only)
Metadata Modified timestamp, record version ID

Monitoring OData

Monitor Transaction Purpose
Service Status /IWFND/MAINT_SERVICE Service status, access logs
OData Logs /IWFND/MONITOR_LOG Call logs, performance, errors
External Monitoring AppFlow/ADF Job logs, row counts, delta status

Quick Reference: Key Transactions

  • /IWFND/MONITOR_LOG - OData monitoring
