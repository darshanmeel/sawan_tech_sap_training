# SAP Note 3255746 — ODP RFC Usage No Longer Permitted

## Summary

SAP Note 3255746 is the single most important compliance note for SAP data extraction. It declares that use of ODP (Operational Data Provisioning) RFC function modules by customers or third-party applications is **no longer permitted**. The note was originally published in 2022, significantly updated in February 2024 (version 4), and revised again in July 2024.

The impact is broad: virtually every third-party ETL connector that extracts from SAP via ODP uses the RFC transport internally. This affects Azure Data Factory, Qlik, Google Cloud Data Fusion, Informatica, Matillion, Fivetran, and many others. The note does not affect SLT, ODP via OData, or SAP-native tools.

## What Is Affected

The note prohibits third-party and customer-developed tools from calling ODP RFC modules directly. Affected tools include:

- **Azure Data Factory** — SAP CDC connector
- **Qlik Replicate** — ODP-based SAP source
- **Google Cloud Data Fusion** — SAP ODP connector
- **Informatica** — ODP connector
- **Matillion** — ODP connector
- **Fivetran** — ODP connector
- Any custom ABAP program or RFC call targeting ODP function modules

## What Is Still Compliant

| Path | Status | Notes |
|---|---|---|
| ODP via OData API | **Compliant** | Slower (~10× vs RFC), but SAP-approved |
| SLT replication | **Compliant** | Not affected by this note at all |
| ABAP CDS views via OData | **Compliant** | Standard-compliant path |
| Direct RFC_READ_TABLE | **Compliant (with caveats)** | See [[note_382318]] — not for production use |
| SAP Datasphere | **Compliant** | SAP's preferred strategic direction |
| SAP-certified connector vendors | **Compliant** | SNP Glue, Theobald via OData, etc. |

## Legal and Practical Enforcement

The note itself is not a court order. However:

- SAP can and does include ODP restrictions in **contract renewals**
- SAP has threatened audits of unlicensed/unpermitted usage
- Large enterprises are taking the risk seriously — legal teams have flagged this
- Contract renewal is the enforcement mechanism, not immediate shutdown

[SME_KNOWLEDGE] Most enterprises using ADF SAP CDC or Qlik ODP are quietly aware of this risk and are evaluating alternatives. The note gives SAP leverage at renewal time.

Source: `raw/notes/SAP_NOTES_REFERENCE.md`

## Practical Alternatives (in Priority Order)

1. **ODP via OData** — compliant, works with most ERP versions, ~10× slower than RFC. Suitable for lower-volume tables or less time-sensitive loads.
2. **SLT (SAP Landscape Transformation Replication Server)** — not affected, real-time capable, separate license required. Best for large-volume or real-time use cases.
3. **SAP Datasphere** — SAP's preferred strategic path. Expensive. Most enterprises treat this as a future migration target.
4. **SAP-certified connectors via OData** — Theobald (via OData mode), SNP Glue. Vendor has taken the compliance risk.
