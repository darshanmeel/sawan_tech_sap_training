# ISSUE-011: Deploy to GitHub Pages, Verify, Submit Sitemap

**Phase:** Launch
**Estimated Effort:** 2-3 hours
**Depends on:** All previous issues
**Skill files:** `SKILLS/deployment.md`, `SKILLS/seo-implementation.md`

---

## Goal

Publish Stage 1 MVP to GitHub Pages, verify everything works, submit to search consoles.

---

## Steps

### Part A: GitHub Pages Configuration

1. Push all code to `main` branch
2. In repository **Settings → Pages**:
   - Source: Deploy from a branch
   - Branch: `main` / folder: `/docs`
3. Save — first deploy starts automatically
4. Wait ~1-2 minutes
5. Visit `https://<username>.github.io/sap-extract-academy/` and verify the landing page loads

### Part B: Sitemap & Robots

1. Verify `docs/sitemap.xml` exists and contains all pages (auto-generated by `build.js` or manually maintained)
2. Verify `docs/robots.txt` exists with AI crawler allows per `SKILLS/seo-implementation.md`
3. Both should be accessible at:
   - `https://<username>.github.io/sap-extract-academy/sitemap.xml`
   - `https://<username>.github.io/sap-extract-academy/robots.txt`

### Part C: Search Console Submission

**Google Search Console:**
1. Sign in to `search.google.com/search-console`
2. Add property → URL prefix
3. Use the GitHub Pages URL (or custom domain if live)
4. Verify via HTML tag (add meta tag to `templates/base.html` and redeploy)
5. Once verified, submit sitemap: `sitemap.xml`

**Bing Webmaster Tools:**
1. Sign in to `bing.com/webmasters`
2. Import from Google Search Console (fastest) OR manual add
3. Submit sitemap

### Part D: Analytics

If Plausible is ready:
1. Uncomment the Plausible script in `templates/base.html`
2. Update `data-domain` to the deployed domain
3. Rebuild and redeploy
4. Verify tracking in Plausible dashboard

If not ready: leave commented. Add to Open Questions.

### Part E: Verification Checklist

Run through every page type:

- [ ] Landing page loads and looks right
- [ ] Tables index shows all 5 tables
- [ ] Each table detail page loads (`/tables/vbak/`, `/tables/lfa1/`, etc.)
- [ ] Each walkthrough loads (15 total)
- [ ] Glossary index shows 32 terms
- [ ] Each glossary term page loads
- [ ] Articles index shows 3 articles
- [ ] Each article loads
- [ ] About, Roadmap, Start pages load
- [ ] 404 page loads for a nonexistent URL
- [ ] Sitemap and robots.txt accessible

### Part F: Lighthouse Audit

For each unique page type (landing, table, walkthrough, article, glossary term), run Lighthouse Mobile and Desktop:

- Performance ≥ 95
- Accessibility ≥ 95
- Best Practices ≥ 95
- SEO ≥ 95

If any page falls short, document in Open Questions and fix before considering launch complete.

### Part G: Cross-Browser Check

Manually verify on:
- Chrome latest (desktop + mobile)
- Firefox latest
- Safari latest (desktop + iOS if possible)

Look for: layout breaks, missing styles, broken interactions.

### Part H: Structured Data Validation

Run every page type through Google's [Rich Results Test](https://search.google.com/test/rich-results):

- Landing page
- A table detail page
- A walkthrough page
- An article
- A glossary term

All should pass with no errors. Warnings acceptable but note them.

### Part I: Write the Announcement

Prepare a launch announcement for:
- LinkedIn personal post
- Reddit r/SAP and r/dataengineering
- Hacker News (Show HN)
- Product Hunt (optional, may be too early)

Announcement template provided in Open Questions for human to finalize.
