Overview

SAP Landscape Transformation (SLT) is the recommended method for real-time, trigger-based replication of SAP tables. It creates database triggers on source tables to capture changes and transfers them to target systems in near real-time.

Part 1: Setup & Enablement Check

Prerequisites

  • SAP source system with DMIS add-on installed
  • SAP LT Replication Server available
  • Sufficient background and dialog work processes
  • User role: SAP_IUUC_REPL_ADMIN

Step 1: Create RFC Connection

Transaction: SM59
Purpose: Connect SLT Replication Server to SAP source system
Configuration:
  • Connection Type: 3 (ABAP to ABAP)
  • Target Host: Source SAP system hostname
  • System Number: Source system number
  • Client: Source system client (e.g., 100)
  • Logon User: Communication user with appropriate roles

Step 2: Open SLT Cockpit

Transaction: LTRC
This is your main SLT configuration and management interface.

Step 3: Create Configuration

  1. Click "Create Configuration" or "New Configuration"
  2. Specify Configuration Name (e.g., "VBAK_Production_Replication")
  3. Add Description
  4. Select RFC Connection to source system
  5. Specify Target System:
    • DB Connection (SAP HANA)
    • ODP / SAP Data Intelligence
    • Other database connections
  6. Define Transfer Settings:
    • Initial Load Mode: Performance Optimized
    • Data Transfer Jobs: 4-8
    • Initial Load Jobs: 4-8
    • Replication Mode: Real Time
  7. Save (generates Mass Transfer ID - MT_ID)

Part 2: Using SLT for Table Replication (VBAK Example)

Step 1: Navigate to Table Selection

Transaction: LTRC
Steps:
  1. Select your configuration
  2. Go to "Table Overview" or "Data Provisioning" tab

Step 2: Select VBAK Table

  1. Click "Data Provisioning" button
  2. Enter Table Name: VBAK (Sales Document Header)
  3. Click "Add" or "Continue"

Step 3: Configure Load Strategy

Initial Load Settings:
  • Reading Type: 1 (Range Calculation)
  • Parallel Read Jobs: 4-8
  • Package Size: 50,000 records
  • Commit Frequency: Every 100,000 records
Delta Load Settings:
  • Replication Mode: Real Time
  • Delta Retrieval: 5,000 records per batch
  • Enable Logging Table: YES

Step 4: Start Replication

  1. Select "Start Replication (Including Initial Load)"
  2. Confirm the action
  3. Monitor progress in LTRC

SLT Automatic Backend Processes

What Happens Behind the Scenes:

1. Create Target Table Structure

Generates VBAK table in destination system with all field definitions and constraints

2. Create Logging Table in Source

Table Name: VBAK#0 stores key field values of changed rows and maintains change sequence

3. Create Database Triggers on VBAK

Three triggers created:

  • INSERT Trigger: Captures new sales documents
  • UPDATE Trigger: Captures modifications
  • DELETE Trigger: Captures deletions
4. Execute Full Initial Load

Reads all existing VBAK records and transfers to target system in 50,000-record batches with data integrity validation

5. Begin Delta Replication

Processes changes from logging table, retrieves 5,000 records per batch, transfers deltas in real-time

SLT Target Destinations

  • SAP HANA Database: Native tables with real-time analytics
  • ODP Framework: Queue Alias with multiple subscribers
  • SAP Data Intelligence: Cloud-based replication
  • Third-Party Databases: Oracle, SQL Server, PostgreSQL
  • Cloud Platforms: S3, BigQuery, Azure Data Lake via Data Intelligence

Quick Reference: Key Transactions

  • SM59 - Create/test RFC connections
  • LTRC - SLT Cockpit (main interface)
  • LTRS - Advanced SLT settings
  • LTRO - Global landscape monitoring

Troubleshooting Tips

Issue: RFC Connection Failed

Check SM59 configuration and system connectivity. Verify hostname and port.

Issue: Insufficient Work Processes

Monitor transaction SM50 and allocate more background jobs as needed.

Issue: Initial Load Slow

Verify Reading Type 1 is selected and increase parallel job count.
